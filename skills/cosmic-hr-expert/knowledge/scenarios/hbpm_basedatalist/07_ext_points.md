## 一、场景专属类（2 个 · 全部禁继承）

### 1.1 PositionBasedataEdit — 编辑视图预置数据保护插件

| 属性 | 值 |
|---|---|
| **全限定类名** | `kd.hrmp.hbpm.formplugin.web.basedata.PositionBasedataEdit` |
| **父类** | `HRDataBaseEdit`（白名单合规） |
| **Jar** | `hrmp-hbpm-formplugin-1.0.jar` |
| **注册顺序** | #6（在 HRHiesButtonSwitchPlugin #5 之后） |
| **生命周期方法** | `afterBindData` |
| **ISV 继承** | ❌ **禁继承**（PR-001）|

**afterBindData 职责（反编译实证）**：
- 读取 `issyspreset` 字段（Boolean）
- 为 `true` → `setBillStatus(BillOperationStatus.VIEW)` → 整页变为只读
- 为 `false` → 不执行任何操作，保持默认可编辑状态

**注意**：同名 `PositionBasedataEdit` 也用于 `hbpm_reportcoreltype` 场景（共用类 · 相同逻辑）。

**ISV 扩展替代方案**：
- 禁止继承 `PositionBasedataEdit`（PR-001 · 场景专属）
- 改用 `HRDataBaseEdit` 或 `HRCoreBaseBillEdit` 并列挂新 FormPlugin
- ISV 的 afterBindData 必须先判断 issyspreset，预置数据跳过自定义逻辑
